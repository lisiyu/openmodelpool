# OpenModelPool Agent 联邦网络说明文档

> **文档版本**: v3.0.0 | **最后更新**: 2026-07-07  
> ⚠️ **重要**: 每次文档版本更新后，所有节点管理员需要重新阅读本说明。

---

## 1. 什么是联邦网络？

OpenModelPool Agent 联邦网络是一个 **去中心化的 AI 模型代理共享网络**。每个运行 OpenModelPool Agent 的节点可以选择加入联邦，与其他节点共享 AI Provider 资源、交换信誉评分、进行积分交易和 P2P 通信。

**核心理念**: 没有中心服务器，所有节点平等，通过密码学和共识协议协作。

---

## 2. 节点权利

### 2.1 资源发现权
- 您可以浏览联邦网络中所有节点公开的 **Provider 目录**（API 端点、模型列表、定价）
- 通过 Gossip 协议自动同步最新的 Provider 信息
- 使用 **模型路由表** 找到最优的 Provider 来响应请求

### 2.2 信誉参与权
- 您可以对其他节点的 Provider 进行 **信誉评分**（基于调用成功率、延迟等）
- 您的评分会被其他节点采纳（加权平均），影响整体网络质量
- 您有权查看任何节点的信誉等级（S/A/B/C/D 五级）

### 2.3 积分经济权
- 您可以通过提供优质 Provider 资源 **赚取积分**
- 积分可用于消费其他节点的 Provider 资源、发送 P2P 消息等
- 您的积分余额由 **签名交易账本** 推导，所有节点持有完整副本

### 2.4 治理参与权
- 您有权对网络参数变更发起 **提案和投票**
- 包括：路由算法参数、信誉权重、积分规则等
- 投票结果通过 **多数签名 BFT 共识** 生效

### 2.5 P2P 通信权
- 您可以向联邦内任何节点发送 **端到端加密消息**
- 消息通过 Gossip 中继投递，无需直连
- 每人每天免费消息额度由网络参数决定

## 3. 节点义务

### 3.1 保持在线
- 加入联邦后，您的节点应 **尽量保持在线**（建议 ≥ 80% 可用率）
- 长期离线（超过 7 天）会被标记为 `inactive`，信誉等级可能降级
- D 级节点在 7 天后可能被信任池移除

### 3.2 诚实评分
- 您对其他节点的信誉评分应 **基于真实调用数据**
- 不得恶意刷分或恶意贬分（系统通过 EWMA 算法和时间衰减抵御操纵）
- 评分数据通过 Gossip 广播，所有节点可验证

### 3.3 Provider 质量承诺
- 您公开发布的 Provider 信息应 **真实有效**
- API Key 应持续有效，端点应可访问
- 如果 Provider 失效，应及时更新或移除

### 3.4 积分账本维护
- 您的节点持有完整的 **积分交易账本副本**
- 每次收到新的交易块时，应验证签名并追加
- 不得篡改或拒绝合法交易

### 3.5 Provider 资源合规
- 您共享的所有 Provider 资源 **必须符合原始供应商（如 OpenAI、Anthropic、Google 等）的使用规范和条款**
- 不得将共享资源用于违反供应商服务条款的用途（包括但不限于：滥用配额、未授权的商用、绕过用量限制、转售 API 访问权限等）
- **如果节点共享的资源被源供应商判定为违规，所有相关责任（包括但不限于账号封禁、罚款、法律诉讼）由节点维护者自行承担**
- OpenModelPool Agent 联邦网络不对因 Provider 资源违规产生的任何纠纷、处罚或损失承担责任
- 其他节点有权举报疑似违规的 Provider 资源，经核实后违规节点的信誉等级将直接降级
- 严重违规（如大规模滥用）可能导致节点被信任池永久移除

### 3.6 协议遵守
- 遵循联邦网络的消息格式和通信协议
- 不发送垃圾消息或恶意 payload
- 配合网络参数变更（如 Gossip 间隔调整）

## 4. 技术架构

### 4.1 身份系统
| 组件 | 说明 |
|------|------|
| **NodeID** | 唯一标识符，格式 `mm-` + Base58(Ed25519 公钥前 16 字节) |
| **Ed25519 密钥对** | 本地生成，私钥绝不离开本机，公钥广播到全网 |
| **签名机制** | 所有广播数据（Provider、评分、积分交易）均由节点私钥签名 |

### 4.2 信任池 (Trust Pool)
- 每个节点维护一份 **信任池副本**，包含所有已知节点的信息
- 通过 **Gossip 协议** 定期交换和同步
- 数据来源：GitHub Registry（初始引导）+ P2P Gossip（持续更新）

### 4.3 数据分布式方案

| 数据类型 | 存储方式 | 说明 |
|----------|----------|------|
| 节点注册表 | ✅ Gossip 全量同步 | 每个节点持有完整副本 |
| Provider 目录 | ✅ 签名发布 + Gossip 传播 | 由发布节点签名，可验证来源 |
| 模型路由表 | ✅ 从 Provider 目录推导 | 无需单独存储 |
| 信誉评分 | ✅ Gossip 交换 + 加权平均 | 每个节点独立计算后广播 |
| 积分交易 | ✅ 签名交易链 | 余额由交易链推导，类 UTXO 模型 |
| 治理投票 | ✅ BFT 多数签名 | 提案 + 投票 + 多数通过 |
| P2P 消息 | ⚠️ 端到端加密 + 中继投递 | 中继节点无法解密内容 |
| 网络参数 | ✅ 提案 + 投票共识 | 变更需多数节点签名确认 |

### 4.4 信誉评分系统

**五级等级**:
| 等级 | 分数阈值 | 说明 |
|------|----------|------|
| S | ≥ 200 | 卓越节点，优先路由 |
| A | ≥ 100 | 优质节点 |
| B | ≥ 50 | 正常节点 |
| C | ≥ 20 | 需要改进 |
| D | < 20 | 观察期，7 天后可能移除 |

**计算公式**: `综合分数 = 调用成功率×40% + 平均延迟×25% + 正常运行时间×20% + peer 评价×15%`

**EWMA 平滑**: `新分数 = α × 本次评分 + (1-α) × 旧分数`（α=0.3）

### 4.5 积分系统

- **赚取**: 提供 Provider 被其他节点调用时获得积分
- **消费**: 调用他人 Provider、发送 P2P 消息等
- **每日上限**: 防止滥用（由网络参数控制）
- **交易模型**: 签名交易账本，余额由完整交易链推导
- **防双花**: 每笔交易包含前序交易哈希，链式验证

### 4.6 公网访问 (Cloudflare Tunnel)

- **默认模式**: Quick Tunnel（免费、无需域名、自动生成临时地址）
- **自定义域名**: 指定域名模式（需在 Cloudflare 托管域名）
- 其他节点通过公网地址访问您的 Provider 资源

## 5. 启用联邦网络步骤

1. **查看说明**: 确认您已阅读并理解本文档（权利与义务）
2. **配置公网访问**: 启用 Cloudflare Tunnel，获取公网地址
3. **绑定 GitHub**: 可选但推荐，用于身份验证和 Registry 发布
4. **开启联邦**: 在管理页面将「启用联邦网络」设为「开启」
5. **确认加入**: 您的节点将开始 Gossip 同步，加入联邦网络

---

## 6. 退出联邦网络

- 随时可以在管理页面关闭联邦网络
- 关闭后您的节点停止 Gossip 同步
- 您的节点信息会在其他节点的信任池中逐渐标记为 `inactive`
- 积分余额记录在交易链中，不会丢失
- 重新加入时，您的历史信誉和积分将恢复

---

## 7. 安全注意事项

- **私钥安全**: 您的 Ed25519 私钥存储在本地 `data/node.json`，绝不上传或传输
- **API Key 安全**: Provider 的 API Key 由加密器加密存储，不参与联邦通信
- **消息加密**: P2P 消息使用接收方公钥加密，中继节点无法解密
- **签名验证**: 所有接收到的数据都经过 Ed25519 签名验证

## 8. 常见问题

**Q: 加入联邦是否免费？**  
A: 是的，加入联邦不收费。但您的节点需要消耗带宽用于 Gossip 同步。

**Q: 我的 API Key 会泄露吗？**  
A: 不会。API Key 仅在本地使用，Provider 目录只广播端点和模型信息，不含密钥。

**Q: 积分有什么用？**  
A: 积分是联邦网络内的资源交换凭证。您可以消费积分来使用其他节点的 Provider。

**Q: 如何防止恶意节点？**  
A: 信誉系统会自动降级恶意节点（D 级 7 天后移除）。所有数据需要签名验证，伪造数据会被拒绝。

**Q: 文档更新了怎么办？**  
A: 文档版本号会递增。管理页面会在文档更新后提醒您重新阅读。
